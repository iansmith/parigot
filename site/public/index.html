parigot: the operating environment for microservices
It's WASM and protocol buffers all the way down! Everything is specified using protocol buffers IDL.
Predictable and repeatable test results, even with dozens of services. We hate that flaky test suite problem too.
Everything is a service. Or you can combine services as you want. Go ahead, make a monolith, it's fine with us.
Pre-built dev container for visual studio. We are so all over that. Never read installation.md again.
Run your code in product cluster, you know, like, for realsies. If you want, we can host your services for production.
Develop and test using just your local machine with a single process. No need to have a giant cluster management tool on your laptop.
Easy and free deployment to our servers when you want to show your friends. Yeah, yeah, yeah... settle down, it works with github actions.
Plenty of built-in services to get you started like databases, files, and authentication. Or you can roll your own, if you are like that, and drop yours in using the same interfaces.
Linux is way cool, but you've been there and done that. (Do you really need four kinds of semaphore system calls?) We are building parigot to be the new hotness.
did we forget to mention it is free and open source?
Ok, ok, cut us some slack. You can try out our hosting service for free, but eventually we charge for that. There is no free lunch--at least not on the internet.
